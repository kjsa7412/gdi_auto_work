# 기능정의서 작성 가이드

## 개요
`screen_spec_template.md`를 복사하여 단위화면별 기능정의서를 작성합니다.
작성된 기능정의서는 PPT 대신 analyze 입력으로 사용됩니다.

## 파일 명명 규칙
```
{화면ID}_spec.md
```
예: `SEA010_spec.md`, `SEA011_spec.md`, `PSB010_spec.md`

## 배치 경로
```
work/task/1.Prep/{화면ID}_spec.md
```

## 작성 원칙

### 1. 필수 섹션
모든 화면에 반드시 포함해야 하는 섹션:
- **1. 화면 개요 (META)** — 화면 식별 정보
- **5. 버튼 (BUTTON)** — 버튼별 기능 (버튼이 1개라도 있으면)

### 2. 화면 유형별 필수 섹션

| 화면 유형 | 필수 섹션 |
|----------|----------|
| list (검색+그리드) | META, SEARCH, GRID, BUTTON |
| list-detail (검색+그리드+상세폼) | META, SEARCH, GRID, FORM, BUTTON |
| form (상세 등록 폼) | META, FORM, BUTTON |
| popup (팝업 그리드) | META, GRID, BUTTON |

### 3. 기술 수준
- **버튼 기능**: 가능한 한 구체적으로 기술 (어떤 테이블에 어떤 동작을, 어떤 조건에서)
- **코드타입**: 알고 있으면 코드타입코드 기재, 모르면 `[코드확인필요]` 표기
- **테이블/컬럼**: 알고 있으면 기재, 모르면 비워두기 (analyze에서 매핑 시도)
- **알림/메시지**: 사용자에게 보여줄 메시지가 있으면 구체적으로 기재

### 4. 해당 없는 섹션
- 해당 없는 섹션은 **삭제**하세요 (빈 테이블로 남기지 마세요)
- `[선택]` 표시된 섹션은 해당 기능이 있을 때만 작성

### 5. 여러 화면 간 연계
- 화면 이동/연계(NAV) 섹션에서 관련 화면 ID를 명시
- 호출 원화면이 있으면 화면 개요에 기재
- 상태별 제어가 화면 간에 걸쳐있으면 양쪽 화면 모두에 기재

## analyze와의 관계
| 기능정의서 섹션 | analyze 분류 태그 |
|---------------|-----------------|
| 1. 화면 개요 | META |
| 2. 검색 영역 | SEARCH |
| 3. 그리드 | GRID |
| 4. 상세 폼 | FORM |
| 5. 버튼 | BUTTON |
| 6. 알림/메시지 | VALID |
| 7. 상태별 제어 | EVENT |
| 8. 화면 이동/연계 | NAV |
| 9. 첨부파일 | FORM (하위) |
| 10. 팝업/모달 | MODAL |
| 11. 특이사항 | EVENT / 기타 |
